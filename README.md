# tournament-ui

Фронтенд турнира на Vue 3 + Vite. Dev-сервер поднимается на порту `5173`.

## Как запустить

### 1. Узнать свой локальный IP

```bash
ipconfig getifaddr en0
```

Если `en0` пустой (например, подключение по кабелю) — попробуй `en1`.
Дальше в примерах используется IP `192.168.1.73` — подставляй свой.

### 2. Прописать IP во фронтенде — `.env`

Файл: `tournament-ui/.env`

```env
SERVER_BASE_URL='http://192.168.1.73:8080'
VITE_API_BASE_URL=http://192.168.1.73:8080/api/v1
```

Старые IP можно не удалять, а закомментировать через `;` (как уже сделано в файле).

### 3. Прописать IP в бэкенде — `application.yml`

Файл: `tournament-ss/src/main/resources/application.yml`

Добавить свой адрес в список разрешённых CORS-origin (порт **5173**):

```yaml
      - "http://192.168.1.73:5173"
```

### 4. Запустить

Бэкенд `tournament-ss` — из IDE (или как обычно запускаешь Spring Boot).

Фронтенд:

```bash
npm install   # только при первом запуске / после смены зависимостей
npm run dev
```

Открыть в браузере: http://localhost:5173

## Полезное

| Команда | Что делает |
|---|---|
| `npm run dev` | Запустить dev-сервер |
| `npm run build` | Собрать прод-сборку |
| `npm run preview` | Посмотреть прод-сборку локально |
| `npm run type-check` | Проверка типов |
| `npm run generate-types` | Сгенерировать типы из OpenAPI (нужен поднятый бэк на `localhost:8080`) |

## Требования

- Node.js `^20.19.0` или `>=22.12.0`
